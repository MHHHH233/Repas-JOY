================================================================================

BASE URL: http://localhost:8000/api (or your domain)

  - Authorization: Bearer {token}
  - Accept: application/json
  - Content-Type: application/json

================================================================================
                        1. AUTHENTICATION ENDPOINTS
================================================================================

1.1. REGISTER USER
----------------------------------------
Method: POST
URL: /api/register
Auth: Not Required

Request Body:
{
  "username": "johndoe",
  "email": "john@example.com",
  "password": "password123",
  "password_confirmation": "password123",
  "address": "123 Main St",
  "date_naissance": "1990-01-01",
  "role": "user"  // optional: "user" or "admin", defaults to "user"
}

Success Response (201):
{
  "success": true,
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": 1,
      "username": "johndoe",
      "email": "john@example.com",
      "address": "123 Main St",
      "date_naissance": "1990-01-01",
      "role": "user",
      "created_at": "2024-01-01T00:00:00.000000Z",
      "updated_at": "2024-01-01T00:00:00.000000Z"
    },
    "token": "1|xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "token_type": "Bearer"
  }
}

Error Response (422):
{
  "success": false,
  "message": "Validation errors",
  "errors": {
    "email": ["The email has already been taken."],
    "password": ["The password confirmation does not match."]
  }
}

================================================================================
                        4. PAGINATION DETAILS
================================================================================

All GET endpoints that return lists support pagination with the following
query parameter:

  - per_page: number (default: 10) - Number of items per page

Pagination Response Structure:
{
  "success": true,
  "data": {
    "current_page": 1,           // Current page number
    "data": [...],               // Array of items
    "first_page_url": "...",     // URL to first page
    "from": 1,                   // Starting item number
    "last_page": 5,              // Last page number
    "last_page_url": "...",      // URL to last page
    "links": [                   // Navigation links
      {
        "url": null,
        "label": "&laquo; Previous",
        "active": false
      },
      ...
    ],
    "next_page_url": "...",      // URL to next page (null if last page)
    "path": "...",               // Base path
    "per_page": 10,              // Items per page
    "prev_page_url": null,       // URL to previous page (null if first page)
    "to": 10,                    // Ending item number
    "total": 50                  // Total number of items
  }
}

Example Pagination Requests:
  - /api/admin/users?per_page=20
  - /api/user/repas?per_page=5
  - /api/admin/categories?per_page=15

================================================================================
                        6. TESTING NOTES
================================================================================

1. AUTHENTICATION:
   - First, register or login to get a token
   - Use the token in the Authorization header: "Bearer {token}"
   - All protected endpoints require this token

2. PAGINATION:
   - Use ?per_page={number} query parameter to control items per page
   - Default is 10 items per page
   - Pagination info is included in the response

3. REQUEST METHODS:
   - GET: Retrieve data
   - POST: Create new resource
   - PUT: Update entire resource (all fields)
   - DELETE: Delete resource

4. VALIDATION:
   - All required fields must be provided
   - Check validation rules in controllers
   - Validation errors return 422 status code

5. SPECIAL BEHAVIORS:
   - User Commande POST: Automatically decrements repas quantity
   - User Commande DELETE: Automatically restores repas quantity
   - User Repas by Category: Only returns repas where onView = true
   - Admin Repas Toggle Visibility: Toggles onView field
   - Commande id_user: Optional field - can be omitted for walk-in customers without accounts

6. TESTING TOOLS:
   - Postman
   - cURL
   - Insomnia
   - Thunder Client (VS Code extension)
   - HTTPie
